## üé≠ Design Principles

### Visual Hierarchy
1. **Lightning/Energy** = Primary focus (yellow/white)
2. **Bronze/Gold** = Secondary elements (frames, borders)
3. **Electric Blue** = Accent/energy (glows, nodes)
4. **Dark Stone** = Background/foundation

### Animation Guidelines
- **Glow effects**: 2-3 second pulse (opacity 0.3 ‚Üí 0.7)
- **Sparks**: 0.6-1.2 second fade (staggered timing)
- **Energy lines**: Continuous subtle pulse
- **Gems**: 1.5 second pulse (staggered by 0.5s)

### Consistency Rules
- All frames use Greek meander patterns
- All energy effects use electric blue
- All metallic elements use bronze/gold gradient
- All lightning uses yellow/white gradient
- All backgrounds use dark storm palette

## üõ†Ô∏è Technical Notes

### SVG Best Practices
- Use filters for glow effects (feGaussianBlur + feMerge)
- Apply gradients for depth (linearGradient, radialGradient)
- Use stroke-width for borders (not fill)
- Implement animations with `<animate>` tags
- Keep file sizes under 10KB per SVG

### Performance Tips
- Use WebP format for raster images (30% smaller)
- Optimize SVG paths (remove unnecessary points)
- Limit filter complexity (max 3 layers)
- Use CSS animations over SMIL when possible
- Total asset package should be under 500KB

## üí° Design Tips for Designers

1. **Contrast is Key**: Use very dark backgrounds to make lightning pop
2. **Layer Your Glows**: Multiple blur layers create depth (8px + 4px + 2px)
3. **Gradient Direction**: Top-to-bottom for depth, radial for energy
4. **Animation Timing**: Stagger animations by 0.5s for organic feel
5. **Greek Patterns**: Use authentic meander/key patterns for authenticity
6. **Lightning Realism**: Jagged paths with white cores look most authentic
7. **Bronze Warmth**: Mix gold highlights with dark bronze shadows
8. **Blue Energy**: Use cyan (#00d4ff) not pure blue for electric feel
